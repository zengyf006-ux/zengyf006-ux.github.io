# ATLAS X Pro 订单与成交审计中心设计

## 目标

在不创建第二套资金、持仓或成交账本的前提下，将现有核心委托、成交历史、OCO 与退出策略投影成可追溯的执行审计记录，让用户能够解释每一笔操作的生命周期、滑点、手续费和关联策略。

## 设计原则

1. 核心账本仍是 `atlasX.pro.v1`；OCO 与退出策略仍分别以现有存储为准。
2. 审计中心只做只读投影，不修改余额、持仓、委托、成交或预留数量。
3. 对核心账本已删除的撤单，仅保存有限的撤单快照；快照不能参与余额或风险计算。
4. 缺少参考价时明确显示“无参考价”，不得伪造滑点。
5. 不把完整审计信息继续挤入主交易屏；使用账户区独立“审计”页和详情面板。
6. 手机端采用卡片与底部详情流程，触控目标不小于 38px。

## 数据来源

- `atlasX.pro.v1`
  - `orders`：当前等待执行的核心委托。
  - `history`：已成交记录。
  - `positions`：用于从持仓进入相关成交审计。
- `atlasX.pro.advancedOrders.v1`
  - OCO 生命周期、止盈子委托、止损触发与终态。
- `atlasX.pro.exitStrategies.v1`
  - 追踪止损、分批退出及终态。
- `atlasX.pro.executionAudit.v1`
  - 只保存已从核心委托列表移除的撤单快照和审计 UI 偏好；不保存资金、余额或持仓。

## 统一审计记录

每条投影记录包含：

- `id`：稳定审计 ID。
- `sourceType`：`core_order`、`core_fill`、`oco`、`exit_strategy`、`canceled_order`。
- `sourceId`：原始账本或策略 ID。
- `symbol`、`side`、`orderType`、`status`。
- `quantity`、`requestedPrice`、`referencePrice`、`executionPrice`。
- `grossNotional`、`fee`、`slippageBps`、`slippageCost`、`totalExecutionCost`。
- `createdAt`、`completedAt`。
- `strategyId`、`parentOrderId`、`childOrderIds`。
- `timeline[]`：按时间排序的生命周期节点。
- `explanation`：基于真实字段生成的执行说明。

## 滑点口径

- 买入不利滑点：`(executionPrice - referencePrice) / referencePrice`。
- 卖出不利滑点：`(referencePrice - executionPrice) / referencePrice`。
- 以 bps 显示；负值表示优于参考价。
- 缺少有效参考价时为 `null`，UI 显示“无参考价”。
- 滑点成本：`grossNotional × slippageBps / 10000`，仅在参考价有效时计算。

## 时间线

### 核心等待委托

- 创建委托。
- 进入等待撮合。
- 当前状态。

### 核心成交

- 提交或账本记录时间。
- 条件满足或市价执行。
- 成交入账。
- 手续费计入。

### OCO

- 创建策略。
- 止盈腿挂单与止损腿监控。
- 止盈、止损、过期、取消或异常终态。

### 退出策略

- 创建策略。
- 激活或等待激活。
- 部分执行、全部执行、取消、过期或异常。

## 页面结构

### 账户区入口

新增“审计”标签，显示总审计记录数。订单、成交和持仓行提供“审计”入口。

### 审计列表

- 概览：记录数、手续费、平均滑点、异常/风险终态数量。
- 筛选：全部、进行中、已成交、策略、已取消。
- 每行显示交易对、来源、状态、数量、执行价格、费用和时间。

### 详情面板

- 执行摘要。
- 生命周期时间线。
- 费用拆分：手续费、滑点成本、总执行成本。
- 关联关系：父订单、子订单、策略 ID。
- 数据完整性说明。

## 边界与非目标

- 不实现真实交易所订单回报。
- 不伪造部分成交；原始数据没有部分成交时不展示。
- 不计算真实资金费率。
- 不改变现有撮合、余额、持仓、预留协调、OCO 或退出策略逻辑。
- 不把审计快照用于绩效、风险或可用余额计算。

## 验收标准

1. 四种固定视口均可打开审计页与详情。
2. 核心委托、成交、OCO、退出策略均形成统一记录。
3. 参考价存在时滑点 bps 与滑点成本计算正确。
4. 参考价缺失时不伪造滑点。
5. 手续费和总成本拆分正确。
6. 从成交行进入审计详情可定位同一 source ID。
7. 核心撤单后仍保留只读撤单审计快照。
8. 不修改核心账本中的现金、持仓、委托、历史或策略状态。
9. 无横向溢出、无控制台错误、移动端触控目标合格。
