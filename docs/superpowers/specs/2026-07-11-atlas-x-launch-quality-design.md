# ATLAS X 第一版上线质感设计说明

## 目标
把当前交易主屏从“视觉可看”提升为“可直接公开展示的完整第一版”。第一版仍为模拟交易终端，不接真实资金、充值提现或交易所密钥。

## 产品边界
- 继续只做交易核心闭环：交易对、K线、盘口、最近成交、下单、持仓、当前委托、历史成交。
- 不增加链上、跟单、策略商城、闪兑、AI荐币、社区、理财或充值提现。
- 桌面和手机必须使用同一产品逻辑，但采用不同布局。

## 本轮必须解决
1. 所有当前可见控件必须有明确反馈，不能出现“看起来能点但没有任何反应”。
2. 交易对选择器必须可切换 BTC、ETH、SOL，并同步更新价格、币种、K线、盘口和下单文案。
3. 自选、导航、通知、账户、更多菜单、图表设置、全屏、盘口设置、账户标签均需有真实状态或明确的轻量反馈。
4. 模拟下单后必须生成持仓与成交记录；刷新后保留交易对、周期、自选和模拟持仓。
5. 视觉继续保持深色、克制、专业，不使用粒子、玻璃堆叠、夸张渐变或无意义动画。
6. 手机端必须保证 390×844 和 430×932 下无横向溢出、无遮挡、底部安全区正确。

## 交互设计
- 交易对选择：桌面使用锚定式浮层，手机使用底部选择面板；支持搜索与三种交易对。
- 一级导航：当前只开放“交易”；其他入口点击后显示“此模块将在后续阶段开放”，避免无反馈。
- 账户区标签：持仓、当前委托、历史成交切换真实内容。
- 通知和账户：使用轻量弹出层，展示模拟环境说明和当前模拟账户状态。
- 收藏：星标可切换并写入本地状态。
- 图表：EMA/BOLL 状态可切换；全屏使用浏览器 Fullscreen API，失败时给出提示。
- 盘口设置：切换“全部 / 仅卖盘 / 仅买盘”。

## 数据与状态
- 使用内置公开演示数据种子，不依赖后端。
- 本地状态写入 `localStorage`，键名统一使用 `atlasX.v1.*`。
- 任何存储失败不得阻断页面，退化为当前会话状态。

## 验收
- 四种尺寸截图：390×844、430×932、1440×900、1920×1080。
- 自动点击交易对、收藏、账户标签、盘口模式、手机下单面板。
- 检查控制台错误、页面脚本错误、空白页、横向溢出、关键模块可见性。
- 人工审查重点：信息层级、字号、空白、按钮状态、弹层比例、手机安全区。
