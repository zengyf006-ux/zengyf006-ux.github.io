# ATLAS X Stage 2 — 移动交易主屏、订单簿与下单流程设计

## 1. 背景与目标

阶段一已经解决公开行情、K 线周期、请求代际、缓存、新鲜度状态和主要交易模块正确性。阶段二只处理移动交易体验的系统级重排，不重写桌面终端，不引入真实资金或真实交易所账户。

目标是把当前“多个桌面模块在手机上分别切换”的体验，升级为一条连续、专业、可审计的移动交易路径：

`市场状态 → 周期与图表工具 → K 线 → 盘口/逐笔 → 下单 → 持仓/委托/审计`

成功标准：

- 390 × 844 与 430 × 932 均无横向溢出、关键遮挡或小于 40px 的主要触控目标。
- 用户从查看价格到提交模拟订单，不需要在多个割裂页面间反复跳转。
- 订单簿具备真实精度聚合、稳定三列、双边/仅买/仅卖模式。
- 下单支持市价、限价、止损市价、止损限价，数量/金额切换和真实盘口派生的成交均价、滑点、手续费预估。
- 全部交易事实继续写入现有 `atlasX.pro.v1` 核心账本，不建立第二套余额、持仓、订单或成交数据源。
- 桌面四区工作台不被阶段二新增控件挤压。

## 2. 已比较方案

### 方案 A：只做 CSS 重排

优点：改动少、交付快。

缺点：无法可靠实现订单簿聚合、订单类型语义、成交均价估算和全屏图表状态；现有 DOM 动态重绘会继续造成选择器和布局竞态。

结论：不采用。

### 方案 B：完全重写移动端 DOM 与交易核心

优点：结构最干净，长期自由度高。

缺点：会复制现有撮合、余额、持仓、OCO、退出、预留、审计和风控逻辑，极易形成第二套事实源并破坏阶段一稳定性。

结论：不采用。

### 方案 C：移动适配层 + 核心交易桥接 + 行情引擎派生视图

优点：保留现有账本和成熟模块；移动端拥有独立布局与状态机；订单簿直接消费 `AtlasMarketDataEngine`；下单只扩展现有核心订单类型，不复制账本。

缺点：需要明确模块边界和启动顺序，并为旧字段提供兼容迁移。

结论：采用。

- 公开行情事实源：`window.AtlasMarketDataEngine`。

## 4. 移动主屏设计

### 4.1 信息层级

手机首屏按以下顺序组织：

1. **市场条**：交易对、最新价、涨跌、实时/缓存/过期状态、收藏、预警、市场筛选。
2. **关键指标条**：24h 高、24h 低、成交额、价差；保持一行，数值使用等宽数字。
3. **周期与工具条**：常用周期横向滚动；指标、绘图、全屏收进单独工具按钮，不与周期争夺宽度。
4. **K 线主视图**：占据当前视口主要高度。
5. **上下文切换条**：盘口、逐笔、持仓三个轻量入口，显示摘要而不是完整模块。
6. **固定交易条**：当前交易对与价格、买入、卖出。

底部现有 `K线 / 盘口 / 成交 / 账户` 导航保留兼容，但视觉降级为二级导航；主交易路径不依赖它才能完成。

### 4.2 轻量蜡烛信息

- 普通模式不再使用大面积浮动详情卡。
- 选中蜡烛后，在周期工具下方显示一条两行信息条：时间、O/H/L/C、涨跌、成交量。
- 点击“更多”才展开完整详情底部层，包含振幅、成交额、EMA、数据源和收盘状态。
- 再次点击同一蜡烛、Esc、关闭、切周期、切交易对均清除选择。
- 桌面继续使用现有完整详情，不受影响。

### 4.3 全屏图表

- 点击全屏后，图表覆盖安全区内整个手机视口。
- 顶部保留交易对、价格、退出按钮。
- 底部提供周期、指标和绘图工具层；工具层默认折叠。
- 全屏状态锁定页面背景滚动，保留当前蜡烛选择和缩放窗口。
- Esc、退出按钮或系统返回语义关闭全屏。

## 5. 专业订单簿设计

### 5.1 输入与聚合

- 输入来自 `AtlasMarketDataEngine.getState().book`。
- 同时兼容数组层级 `[price, quantity]` 与对象层级 `{ price, quantity }`。
- 卖盘按聚合档位向上取整，买盘向下取整，避免价格跨越中间价。
- 同一档位数量相加，再按最优价向外计算累计量。
- 可选档位根据当前价格精度生成，不使用固定 `0.1 / 1 / 10`：例如 BTC 可为 `0.1 / 1 / 10 / 50`，低价资产可为 `0.0001 / 0.001 / 0.01 / 0.1`。

### 5.2 展示

- 三列固定为价格、数量、累计；使用 CSS Grid 与 `font-variant-numeric: tabular-nums`。
- 双边模式：卖盘在上、最新价居中、买盘在下。
- 仅买盘和仅卖盘模式填满可用高度，不保留空白半区。
- 深度背景按本侧最大累计量计算，不跨买卖盘共用比例。
- 点击任一价格继续写入现有下单价格输入，不直接提交订单。
- 数据过期时保留最后有效深度但明确标注“已过期”，不得伪装实时。

## 6. 下单流程设计

### 6.1 订单类型

核心订单类型规范化为：

- `market`：市价单，立即按当前模拟市场执行。
- `limit`：限价单，等待价格满足后按限价执行。
- `stop_market`：止损市价单，触发后按触发时市场价执行。
- `stop_limit`：止损限价单，先触发，再按限价条件等待执行。

兼容规则：旧 `stop` 在读取时视为 `stop_market`，不丢失旧订单。

### 6.2 止损限价状态

`stop_limit` 生命周期：

1. `waiting_trigger`：等待触发价。
2. `triggered`：触发完成，开始作为限价单等待。
3. `filled` 或用户撤单。

买入止损限价：市场价上穿触发价后激活；激活后当市场价小于等于限价时成交。

卖出止损限价：市场价下穿触发价后激活；激活后当市场价大于等于限价时成交。

### 6.3 数量/金额模式

- 顶部提供“按数量 / 按金额”分段控件。
- 主输入只显示当前模式；另一个字段保留在 DOM 中并实时同步，保证旧模块和测试兼容。
- 卖出时“按金额”根据预计执行价反算数量，并受统一可用持仓限制。
- 百分比快捷键作用于当前可用余额或统一可用持仓。

### 6.4 执行预估

市价单预估直接遍历当前订单簿：

- 买入消耗卖盘，卖出消耗买盘。
- 计算可覆盖数量、VWAP、参考价、方向一致的不利滑点 bps、滑点成本、手续费和总成本。
- 深度不足时显示覆盖比例和未覆盖数量，并阻止把“预计全部成交”作为文案。
- 限价与止损限价使用委托价作为预计价，滑点显示为 `--`，不伪造队列成交概率。
- 止损市价在未触发前使用当前深度作“触发时仅供参考”的估算，并明确标注。

预估只读，不修改账本，不参与实际模拟成交定价；实际成交继续由核心交易逻辑决定并进入审计。

### 6.5 提交前信息

提交按钮上方固定展示：

- 预计成交均价
- 预计手续费
- 预计滑点 / 无法估算原因
- 盘口覆盖率
- 订单生效条件
- 模拟交易安全声明

大额确认、交易安全锁、只减仓、Post Only、交易计划、OCO、退出策略继续使用既有模块。

## 7. 核心桥接接口

`app.js` 暴露只包含必要能力的只读/命令桥接：

```js
window.AtlasCoreTrading = Object.freeze({
  getState(),
  getMarket(),
  setOrderType(type),
  setSide(side),
  syncOrderFields(source),
  submitOrder(),
  renderOrderTicket(),
  matchOpenOrders(),
});
```

约束：

- `getState()` 返回深拷贝，不暴露可直接修改的核心对象。
- Stage 2 不直接覆盖现金、持仓、委托或成交数组。
- 所有提交仍调用核心 `submitOrder()`。

## 8. 错误与降级

- 行情引擎未就绪：订单簿显示等待状态，下单仍可使用当前页面价格，但市价深度预估显示不可用。
- 深度为空：不伪造 VWAP；显示“盘口深度不足”。
- CSS 或 Stage 2 模块加载失败：保留阶段一现有手机布局，不能造成白屏或阻断桌面。
- 非法订单字段：核心拒绝提交并使用现有 Toast；Stage 2 同步显示字段级错误。
- 旧存储字段：在内存读取时归一化，不批量破坏性迁移。

## 9. 验收

### 移动主路径

- 390 和 430：市场状态、周期、图表、盘口摘要、买卖条均在首屏合理层级。
- 打开/关闭全屏图表，背景滚动锁定与恢复正确。
- 蜡烛轻量信息条不覆盖主要价格区域。
- 从盘口点击价格打开限价下单，价格正确带入。

### 订单簿

- 原始深度聚合后数量守恒。
- 卖盘向上、买盘向下聚合。
- 累计量单调增加。
- 三种模式和精度偏好刷新后保持。
- 无横向溢出，主要控件高度不低于 40px。

### 下单

- 四类订单写入同一核心账本。
- `stop_market` 触发后按市场执行。
- `stop_limit` 触发前不成交，触发后按限价条件成交。
- 数量/金额切换双向同步。
- 市价 VWAP、滑点、手续费和覆盖率计算正确。
- 下单预估不修改现金、持仓、委托、成交、OCO、退出或预留账本。

### 回归

- 真实网关 contract 和 smoke 继续通过。
- 390、430、1440、1920 全链继续通过。
- 桌面布局和功能无 Stage 2 特有视觉变化。
- 无控制台错误、页面错误、白屏或横向溢出。

## 10. 非目标

- 不接真实资金或真实交易所账户。
- 不做永续合约、杠杆、保证金、强平或资金费率。
- 不实现真实交易所撮合队列和部分成交回报。
- 不重写桌面四区工作台。
- 不新增后端账户系统。
