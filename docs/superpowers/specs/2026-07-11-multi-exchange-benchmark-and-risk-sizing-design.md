# 多交易所对标与风险仓位工具设计

## 目标

ATLAS X Pro 不复刻任何单一交易所，而是持续对标 Binance、OKX、Bybit、Coinbase Advanced 与 Kraken Pro，从它们的成熟交易流程、风控反馈、信息密度和跨端交互中抽取优点，过滤广告化、功能堆叠、视觉噪声和移动端拥挤等缺点，形成一套统一的专业模拟交易终端。

本轮交付为“交易计划与风险仓位计算器”，解决用户在下单前需要手工计算风险金额、止损距离、建议数量、最大亏损和盈亏比的问题。

## 对标原则

1. **Binance**：吸收订单区高信息密度、价格/数量/总额联动与高级委托入口；不吸收入口过多、营销模块干扰和复杂层级。
2. **OKX**：吸收交易前风险参数、TP/SL 语义和明确的执行上下文；不吸收重复设置和过多弹层。
3. **Bybit**：吸收仓位计算、止损止盈联动和交易前结果预览；不吸收高风险产品导向或夸张视觉强调。
4. **Coinbase Advanced**：吸收清晰、克制的表单分组和费用透明；不吸收功能覆盖较弱的部分。
5. **Kraken Pro**：吸收专业桌面布局、紧凑数据表达和订单确认逻辑；不吸收学习成本高的晦涩术语。

所有新增功能必须满足：
- 不复制品牌图形、配色、文案或页面结构；
- 只借鉴通用交互模式和业务逻辑；
- 当前仍为现货模拟交易，不伪装成真实交易所；
- 不新增真实资产、密钥、充值、提现或交易所账户连接；
- 桌面和手机端都必须可用；
- 与现有订单保护、账本绩效、组合风险、市场情报和行情路由零回归。

## 本轮功能范围

### 1. 交易计划入口

在现有下单面板的高级选项与订单摘要之间加入一个紧凑的“交易计划”区域。默认折叠，显示当前单笔风险比例和建议数量；展开后显示完整输入与计算结果。

### 2. 输入参数

- 入场价：市价单跟随当前标记价格；限价/止盈止损委托跟随订单价格，可手动刷新。
- 止损价：必填后才产生建议数量。
- 目标价：可选；填写后计算盈亏比。
- 单笔风险：默认账户权益的 1%，允许 0.25%、0.5%、1%、2% 快捷选择和 0.1%–5% 手动输入。

### 3. 核心计算

- 风险金额 = 账户权益 × 单笔风险比例。
- 单位风险 = |入场价 − 止损价| + 预计开仓手续费/单位 + 预计止损平仓手续费/单位。
- 原始建议数量 = 风险金额 ÷ 单位风险。
- 买入建议数量不得超过可用现金可承担的最大数量。
- 现货卖出建议数量不得超过当前币种可卖持仓。
- 预计最大亏损包括价格止损损失与双边预计手续费。
- 目标价存在时，盈亏比 = 预计净盈利 ÷ 预计最大亏损。

### 4. 方向校验

- 买入计划：止损价必须低于入场价；目标价如填写，必须高于入场价。
- 卖出计划：当前为现货减仓，不计算做空风险；建议数量只受持仓上限约束，并显示明确说明。
- 无效参数不自动填单，只显示原因。

### 5. 一键填入

“使用建议数量”按钮将建议数量写入现有 `#orderQuantity`，触发原有数量—总额—手续费—滑点联动，不绕开现有下单校验和订单执行保护。

### 6. 状态与持久化

风险比例、止损价和目标价按交易对保存在独立的本地存储键中；不修改核心模拟账本结构。切换交易对时加载对应计划，避免把 BTC 的价格带到 ETH。

### 7. 视觉与跨端

- 桌面：嵌入订单面板，不增加新的全屏页面；折叠态高度不超过 54px。
- 手机：在底部订单 Sheet 内自然滚动，不遮挡提交按钮；触控按钮最小高度 40px。
- 中文语义文本使用现有 UI 字体，数字结果使用等宽数字字体。
- 不使用大面积渐变、发光、动画或交易所品牌色复制。

## 错误处理

- 无止损、止损方向错误、权益为零、价格为零时，计算器给出明确状态且禁用一键填入。
- 计算数量低于市场最小展示精度时，显示“风险预算过小”。
- 可用现金或持仓限制导致截断时，显示“已按余额/持仓上限调整”。
- 所有计算均为本地模拟，不能表述为投资建议或保证亏损上限。

## 验收标准

1. 四种视口 390×844、430×932、1440×900、1920×1080 全部通过。
2. 固定账户权益、入场价、止损价和风险比例下，风险金额、建议数量、最大亏损和盈亏比与独立测试计算一致。
3. 买入方向错误的止损价必须阻止填入。
4. 买入数量必须受可用现金上限约束。
5. 一键填入后原订单数量、总额和手续费摘要同步更新。
6. 切换交易对后计划参数互不污染。
7. 手机订单 Sheet 无横向溢出，提交按钮仍可到达。
8. 原有完整验收链全部继续通过。

## 后续对标路线

风险仓位工具完成后，按以下顺序继续：
1. 图表交易工具：价格线、止损/止盈线、订单线与持仓成本线；
2. 高级委托：OCO、追踪止损、分批止盈和有效期；
3. 订单与成交审计：执行时间线、滑点归因和费用拆分；
4. 工作区布局：可折叠面板、密度档位和快捷键；
5. 资产与风险：按币种风险预算、相关性和组合风险限额。
