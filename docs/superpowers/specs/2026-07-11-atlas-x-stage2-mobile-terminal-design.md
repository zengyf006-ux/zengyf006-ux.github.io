# ATLAS X Pro Stage 2 — 移动交易主屏、订单簿与下单流程设计

日期：2026-07-11  
状态：已批准连续实施  
基线提交：`416a603da58d1dafc230264e54dfd401976b269d`

## 1. 背景

Stage 1 已解决实时行情、周期正确性、加载空白、K 线详情、高低点和数据新鲜度问题，但手机端仍由桌面模块切片组成：顶部行情、周期栏、图表、盘口、下单、持仓之间缺少统一信息层级；下单面板订单类型不足；订单簿列宽和精度聚合不够专业；K 线详情卡遮挡面积偏大。

本阶段目标不是模仿某一家交易所，而是吸收成熟交易终端的共性：市场状态清晰、关键动作就近、数字稳定对齐、风险与费用在提交前可见、复杂功能按需展开。

## 2. 方案选择

### 方案 A：仅追加 CSS

优点：改动小。  
缺点：无法解决 DOM 顺序、状态所有权、订单类型、数量/金额模式和盘口聚合。  
结论：不采用。

### 方案 B：移动端编排层，复用现有核心账本

新增专门的移动编排模块，重组手机 DOM、维护手机 UI 状态、投影现有行情和模拟交易账本；不创建第二套余额、持仓、订单、成交、OCO、退出策略或审计账本。

优点：保留现有核心正确性，同时可以系统重排体验。  
缺点：需要明确适配旧 DOM 和事件接口。  
结论：采用。

### 方案 C：重写全部 app 与交易核心

优点：理论上最整洁。  
缺点：会重新引入撮合、资金、风险和审计一致性风险，范围不可控。  
结论：Stage 2 不采用。

## 3. 设计原则

1. 手机不是缩小版桌面。
2. 行情、K 线、盘口、下单和账户使用同一行情会话与同一模拟账本。
3. 主屏只放高频信息，低频高级功能进入工具抽屉或折叠区。
4. 所有价格和数量使用等宽数字、固定列宽和一致小数位。
5. 下单前展示成交均价、手续费、滑点、占用余额和风险提示。
6. 不改变 Stage 1 实时行情引擎的所有权。
7. 不改变现有余额、持仓、成交、订单、OCO、退出策略、预留、审计和预警存储键。
8. 手机关键触控目标不低于 44px；紧凑数字行允许 36px，但不能承担主操作。

## 4. 信息架构

### 4.1 手机交易主屏

从上到下固定为：

1. **市场头部**：交易对、模拟环境、实时状态、当前价、涨跌、收藏、预警、数据健康。
2. **市场摘要**：24h 高低、成交额、振幅、点差；默认单行，可展开。
3. **周期与图表工具**：常用周期直接显示，其余周期进入“更多”；指标、绘图、全屏进入工具按钮。
4. **K 线区**：优先占据首屏主要高度。
5. **上下文切换条**：盘口、成交、持仓、委托；切换不离开交易上下文。
6. **底部交易栏**：当前价、买入、卖出；安全区适配。

账户与审计保留独立完整视图，但高频持仓和委托摘要可以在图表下方按需展开。

### 4.2 K 线详情

手机详情由大浮层改为两级：

- 默认：图表上方紧凑 OHLC 条，仅显示时间、O/H/L/C、涨跌和成交量。
- 上滑或点击“详情”：底部轻量抽屉展示振幅、成交额、EMA、数据源、收盘状态。

取消方式：再次点击同一蜡烛、点击空白、关闭、切周期、切市场、进入全屏。

### 4.3 全屏图表

- 使用固定全屏层，不依赖浏览器原生 fullscreen 才能工作。
- 支持横竖屏布局。
- 底部工具抽屉包含周期、指标、绘图、重置、回到最新。
- 退出后恢复原滚动位置、周期、图表偏移和选中状态清空。

## 5. 订单簿设计

### 5.1 数据模型

订单簿仍来自 `AtlasMarketDataEngine` 的同一会话。新增只读聚合函数：

```js
aggregateBook({ bids, asks, tickSize, rows, mode })
```

返回：

```js
{
  bids: [{ price, quantity, cumulative }],
  asks: [{ price, quantity, cumulative }],
  bestBid,
  bestAsk,
  spread,
  spreadBps,
  bidRatio,
  askRatio,
  maxCumulative
}
```

聚合不得修改原始行情状态。

### 5.2 视觉与交互

- 三列固定：价格、数量、累计。
- 价格列右对齐，数量和累计使用稳定宽度。
- 深度背景按累计量绘制，不改变文字对比度。
- 支持双边、仅买盘、仅卖盘。
- 精度选项根据市场价格自动生成，不再固定 `0.1/1/10`。
- 点击价格填入下单面板并打开对应买卖方向，但不自动提交。
- 中间价显示方向、价差和价差 bps。
- 逐笔成交支持暂停滚动、方向筛选和成交量突出。

## 6. 下单流程设计

### 6.1 订单类型

手机和桌面统一支持：

1. 市价
2. 限价
3. 止损市价
4. 止损限价

现有 OCO、追踪止损和分批退出继续作为高级策略区，不塞入基础订单标签。

### 6.2 输入模式

- 数量模式：输入基础资产数量。
- 金额模式：输入 USDT 金额。
- 两者切换保持等值，不清空用户已输入内容。
- 百分比快捷键：25%、50%、75%、100%。
- 百分比必须基于统一可用余额或统一可卖数量，不绕过预留协调器。

### 6.3 预估

提交前统一展示：

- 参考价格
- 预计成交均价
- 预计滑点及 bps
- 预计手续费
- 预计总支出或净收入
- 可用余额/可卖数量
- 订单可能立即成交、等待成交或触发后执行的状态说明

市价单使用当前订单簿逐档估算 VWAP；数据不足时明确显示估算不完整，不伪造精确值。

### 6.4 风险与确认

- 继续使用现有 execution guard、大额确认和交易安全锁。
- 只减仓、Post Only 只在适用订单类型展示。
- 触发价、限价关系错误时阻止提交并给出字段级提示。
- 按钮文案包含动作、资产和订单类型，例如“市价买入 BTC”。
- 模拟环境提示保留，但缩短为非阻塞说明。

## 7. 组件边界

### `mobile-terminal-shell.js`

负责：移动 DOM 编排、上下文标签、滚动位置、底部栏和全屏层。  
不负责：行情请求、撮合、余额或订单存储。

### `professional-orderbook.js`

负责：聚合、精度选项、盘口渲染、点击价格和逐笔成交控制。  
依赖：`AtlasMarketDataEngine` 和现有市场精度信息。

### `professional-order-entry.js`

负责：订单类型、数量/金额模式、订单簿 VWAP 预估、字段校验和提交前摘要。  
提交仍调用现有交易核心公开入口；若现有核心没有稳定入口，则增加薄适配器，不复制撮合逻辑。

### `mobile-chart-workspace.js`

负责：紧凑详情、全屏图表、工具抽屉和恢复上下文。  
依赖：`AtlasChartExperience` 和现有图表渲染。

## 8. 状态与持久化

新增 UI 偏好键：

```text
atlasX.pro.mobileTerminal.v1
```

仅保存：

- 最后上下文标签
- 常用周期
- 盘口模式
- 盘口精度
- 数量/金额模式
- 折叠状态

禁止在该键中保存余额、持仓、委托、成交或风险账本。

## 9. 加载和错误处理

- 缓存行情存在时立即绘制，不显示全屏加载层。
- 切周期、切市场只显示紧凑状态，不清空旧图。
- 盘口暂时缺失时保留框架并显示“等待深度数据”，不生成随机档位。
- 数据 stale/offline 时下单面板明确提示；offline 时阻止市价提交，限价单是否允许进入模拟等待队列由现有执行保护决定。
- 模块初始化失败时退回 Stage 1 手机视图，不能白屏。

## 10. 验收标准

### 手机专项

- 390×844 和 430×932 首屏无横向溢出。
- 市场头部、周期、图表、上下文切换和交易栏均可达。
- K 线详情默认占用不超过图表高度 32%。
- 全屏图表可进入和退出，状态恢复正确。
- 盘口三列在两种手机宽度下不跳列。
- 精度切换后档位聚合正确，总量守恒在允许舍入误差内。
- 盘口点击只填价，不自动下单。
- 四种基础订单类型字段和校验正确。
- 数量/金额切换等值误差小于市场最小精度。
- 市价 VWAP、手续费和滑点预估与测试订单簿一致。
- 预留、OCO、退出策略和审计不被污染。
- 触控目标、软键盘、底部安全区和滚动锁正确。

### 全量回归

继续运行：实时行情、周期、K 线、执行保护、绩效、风险、OCO、退出策略、预留、工作区、预警、审计、情报、筛选器、数据健康、中文字体和四端视觉 QA。

## 11. 非目标

- 不接真实交易所账户。
- 不实现真实充值、提现或钱包。
- 不在本阶段建立模拟永续合约资金费率和杠杆账本。
- 不重写桌面工作区。
- 不机械复制 OKX、Binance、Bybit 或其他品牌视觉。
