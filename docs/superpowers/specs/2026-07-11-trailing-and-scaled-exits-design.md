# 追踪止损与分批止盈设计

## 目标

在 ATLAS X Pro 现有普通订单、风险仓位、图表交易和 OCO 之上增加两类高价值退出策略：追踪止损与三档分批止盈。设计吸收大型专业交易所的退出策略能力，但保持现货模拟终端的清晰边界，不引入杠杆、合约、复杂条件树或满屏配置。

## 独立存储

使用 `atlasX.pro.exitStrategies.v1`：

```json
{
  "version": 1,
  "strategies": []
}
```

不得与 `atlasX.pro.advancedOrders.v1` 的 OCO 记录混存，避免不同策略字段互相污染。

## 追踪止损

### 输入

- 数量；
- 激活价：可选，留空代表立即激活；
- 回撤比例：0.1%–20%；
- 有效期：长期有效、15 分钟、1 小时、当日有效。

### 状态

- `waiting_activation`：等待价格达到激活价；
- `active`：已激活并跟踪最高价；
- `triggering`：触发后正在通过核心执行；
- `completed`：核心市价减仓完成；
- `expired`：到期未触发；
- `canceled`：用户取消；
- `error`：执行失败。

### 计算

- 无激活价时，创建时的当前价作为初始最高价；
- 有激活价时，当前价达到或高于激活价后开始跟踪；
- 最高价只允许上移：`peak = max(peak, current)`；
- 触发价：`trigger = peak × (1 - trailPercent / 100)`；
- 当前价小于或等于触发价时，先将状态改为 `triggering` 以释放自身本地预留，然后通过原订单表单提交只减仓市价卖单；
- 不直接修改现金、持仓或成交历史。

### 本地预留

`waiting_activation` 和 `active` 状态的追踪止损数量计入退出策略预留。普通卖单、OCO 和新退出策略都必须扣除该数量，防止同一现货持仓被重复使用。`triggering` 状态不再计入预留，以允许自身核心卖单通过执行保护。

## 分批止盈

### 输入

- 总数量；
- 三档止盈价格；
- 三档比例，默认 30% / 30% / 40%；
- 有效期：长期有效、15 分钟、1 小时、当日有效。

### 校验

- 当前交易对必须存在可用现货持仓；
- 总数量不得超过扣除核心卖单和活动追踪止损后的可用数量；
- 三档比例必须大于零且总和等于 100%；
- 三档价格必须全部高于当前价并严格递增；
- 每档名义金额不得低于 5 USDT；
- 完全相同的活动策略不得重复创建。

### 创建

- 按三档比例计算数量，最后一档吸收精度余数，确保总数量严格等于输入总量；
- 每档通过原订单表单创建只减仓限价卖单；
- 每个核心订单 ID 写入对应腿；
- 任一腿创建失败时，撤销本次已创建的所有腿并将策略标记为 `error`；
- 不直接伪造核心订单。

### 状态

- `creating`：正在逐腿创建；
- `active`：三腿全部创建；
- `partially_completed`：至少一腿成交，仍有活动腿；
- `completed`：全部腿成交；
- `partially_canceled`：部分腿取消，仍有活动腿；
- `canceled`：所有未完成腿已取消；
- `expired`：到期后剩余腿已撤销；
- `error`：创建或监控异常。

每条腿记录 `pending / filled / canceled`。

## 执行保护整合

`execution-guard.js` 的卖出可用数量改为：

`持仓数量 - 核心卖单冻结 - 活动追踪止损本地预留`

提示文案分别展示核心冻结和退出策略预留。OCO 创建和退出策略表单使用同一可用数量口径。

## UI

在 OCO 面板下方新增折叠式“退出策略”面板，内部用两个页签：

- 追踪止损；
- 分批止盈。

桌面折叠态不超过 50px。手机端进入订单 Sheet 的自然滚动流，按钮最小高度 40px。状态列表仅显示当前交易对最近 5 条，中文状态使用 UI 字体，价格和数量使用等宽字体。

## 图表联动

追踪止损活动时，在图表交易层显示当前触发价；分批止盈的核心订单已经通过现有委托线显示，不重复增加另一套线。

## 验收

1. 创建追踪止损后不产生核心委托，但本地预留数量正确；
2. 普通卖单不能使用被追踪止损预留的数量；
3. 价格上升后峰值和触发价只上移不下移；
4. 跌破触发价后产生原核心卖出成交，持仓按数量减少；
5. 创建三档分批止盈后产生三笔核心只减仓限价卖单，数量合计严格等于总数量；
6. 任一腿创建失败时已创建腿全部回滚；
7. 模拟一腿成交后策略为部分完成，三腿成交后为完成；
8. 到期或取消能撤销剩余核心腿；
9. 切换交易对后状态列表和追踪线隔离；
10. 四种视口与全部既有验收零回归。
