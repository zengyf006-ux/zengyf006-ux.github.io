# OCO 高级委托 Implementation Plan

**Goal:** 在现有现货模拟交易终端中增加只冻结一次持仓的 OCO 保护单与有效期管理。

**Architecture:** 新建独立 `advanced-order-oco.js/.css`。止盈腿通过原订单表单进入核心限价委托；止损腿与有效期保存在独立本地存储。模块监控当前价格、当前委托和成交记录，通过原撤单/提交入口完成取消与止损执行，不直接修改现金、持仓或成交历史。

## Task 1：失败验收

- 新建 `qa/atlas-x-pro/advanced-order-oco.mjs`；
- 固定 BTC 持仓 0.5、现金 50,000；
- 创建 0.05 BTC OCO；
- 验证核心只新增一笔止盈限价卖单、只冻结 0.05；
- 验证独立 OCO 记录与核心订单 ID；
- 通过改写当前显示价格触发止损，验证先撤单再成交、持仓减少；
- 注入匹配成交 DOM 验证止盈完成识别；
- 将过期时间改为过去验证自动撤单与过期状态；
- 验证重复创建阻止、手机无溢出；
- 接入四端流水线并确认红灯只落在新步骤。

## Task 2：OCO 表单与状态存储

- 在风险仓位面板后插入折叠式高级委托面板；
- 输入数量、止盈价、止损触发价、有效期；
- 读取核心持仓与其他卖单冻结量；
- 实时展示可用数量、止盈名义金额、止损距离；
- 使用 `atlasX.pro.advancedOrders.v1` 保存最多 30 条记录；
- 交易对切换后只展示当前交易对记录。

## Task 3：通过原核心创建止盈腿

- 保存当前订单表单上下文；
- 切换卖出、限价、只减仓；
- 写入止盈价与数量并点击原提交按钮；
- 如出现原确认弹窗，完成该次已授权确认；
- 比较创建前后的撤单按钮 ID，记录新增核心订单 ID；
- 恢复原表单上下文；
- 未找到新增订单时将 OCO 标记 `error`。

## Task 4：监控与执行

- 观察 `#lastPrice`、`#ordersBody`、`#historyBody`、`#activePair`；
- 止盈腿消失且存在匹配卖出成交时标记 `completed_take_profit`；
- 止盈腿被手动撤销且无匹配成交时标记 `canceled`；
- 到期时点击对应原撤单按钮并标记 `expired`；
- 止损触发时先撤止盈，等待委托消失，再通过原市价卖出入口执行数量；
- 成交出现后标记 `completed_stop`；失败则标记 `error` 并记录原因；
- 防止同一 OCO 重复执行。

## Task 5：样式、完整验证与人工审查

- 桌面折叠态紧凑；手机按钮至少 40px；
- 状态列表最多显示当前交易对最近 5 条；
- 四端全套流水线通过；
- 下载桌面与 390 手机实图；
- 任何缺陷先加入可复现守卫再修复。
