## 3. 当前已完成能力

### 核心交易终端

- 专业桌面工作区与手机独立交易流程。
- 公开行情优先、断网自动降级演示行情。
- 市场列表、自选、搜索、行情摘要、K 线、订单簿和逐笔成交。
- 市价、限价、高级模拟委托、余额、持仓、委托、成交、费用、滑点和盈亏。
- 下单执行保护、大额确认、交易安全锁和二次解锁。
- 账本派生绩效分析、组合风险中心、风险仓位计算和交易计划。
- 图表交易层、OCO、追踪止损、分批退出和统一预留协调。
- 工作区模式、命令中心、交易预设和快捷键。
- 市场情报、行情路由、数据健康、中文语义字体和高级视觉状态。
- 手机首屏保护和手机账户工具。

### 专业预警中心

- 桌面与手机使用同一个通知按钮、同一个未读徽标和同一套事件状态。
- 旧价格提醒规则自动迁移到专业预警中心。
- 旧提醒入口和旧弹窗已清理，不再维护两套提醒系统。
- 支持价格穿越规则、成交事件、OCO 终态和退出策略终态。
- 手机端规则、全部已读、快捷价格和顶部铃铛触控尺寸已加强。

### 订单与成交审计中心

- 账户区新增独立“审计”标签和详情面板。
- 统一投影：核心等待委托、核心成交、OCO、退出策略和已撤委托快照。
- 展示订单/策略来源、状态、方向、数量、委托价、参考价、执行价、手续费和时间线。
- 支持手续费、滑点 bps、滑点成本和总执行成本拆分。
- 买入和卖出采用方向一致的不利滑点口径；负值代表优于参考价。
- 缺少有效参考价时明确显示“无参考价”，不伪造滑点。
- 从订单、成交和持仓记录进入关联审计详情。
- 撤单前捕获有限只读快照；快照不参与余额、持仓、风险、绩效或预留计算。
- 桌面和手机均支持筛选、详情和完整生命周期查看。

## 4. 订单与成交审计的数据边界

- 核心账本仍为 `atlasX.pro.v1`。
- OCO 账本仍为 `atlasX.pro.advancedOrders.v1`。
- 退出策略账本仍为 `atlasX.pro.exitStrategies.v1`。
- 审计独立存储为 `atlasX.pro.executionAudit.v1`，只保存最多 80 条撤单快照和固定 UI 偏好。
- 审计中心是只读投影，不创建第二套余额、持仓、委托或成交账本。
- 当前模拟成交若没有来源参考价，无法可靠计算滑点归因；不得用当前行情反推历史参考价。
- 当前原始数据没有真实交易所逐笔回报，因此不伪造真实部分成交、撮合队列、资金费率或交易所拒单原因。

## 7. 已知风险与暂停条件

### 已知风险

- 所有订单、策略、审计和提醒状态仍主要保存在当前浏览器本地存储；清理站点数据会丢失。
- 预警只在页面运行期间监控，关闭页面后不会后台继续运行。
- 审计详情依赖现有账本字段完整度；旧记录可能缺少参考价、提交时间或关联 ID。
- 审计模块通过有限频率刷新现有账本；后续数据量明显增大时需评估增量事件机制。
- 目前仍为模拟交易，不得对外宣称已接入真实交易所或真实资金。

### 必须暂停的情况

- 后续部署候选相对 `main` 不是纯快进。
- 后续最终候选的四端验收或视觉 QA 出现红灯。
- 线上路径加载旧版、404、白屏、控制台错误或明显布局溢出。
- 审计功能修改了现金、持仓、成交、委托、OCO、退出策略或预留账本。
- 需要强制更新 `main` 才能部署。
